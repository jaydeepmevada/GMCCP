6) Basic Express server (server/index.js) - summary
- Setup express, cors, bodyParser (express.json)
- Connect to MongoDB using mongoose.connect(MONGO_URI)
- Mount routes: /api/auth, /api/complaints, /api/admin, /api/cities
- Start server on process.env.PORT

7) Mongoose schemas (models)
User:
- name, email (unique), passwordHash, phone?, role (user/admin), preferredLanguage, timestamps

Complaint:
- user (ObjectId), city, title, description, photoUrl, status (submitted/verified/resolved), assignedTo, timestamps

City (config):
- name, emailForCorporation, contactPerson, active(boolean)

EmailLog (optional):
- complaintId, to, subject, body, status, sentAt

8) API endpoints (overview)
Auth:
- POST /api/auth/register  -> register (name,email,password)
- POST /api/auth/login     -> login -> returns JWT

Cities:
- GET /api/cities          -> list cities for dropdown

Complaints (user):
- POST /api/complaints     -> create complaint (multipart/form-data: photo + fields)
- GET /api/complaints      -> list complaints for logged-in user
- GET /api/complaints/:id  -> complaint detail

Admin (protected):
- GET /api/admin/complaints            -> list all complaints (filters: city/status)
- POST /api/admin/complaints/:id/verify -> set status verified + trigger email to city
- POST /api/admin/complaints/:id/send-email -> manual send
- GET /api/admin/email-logs            -> view email logs

9) Authentication flow (backend + frontend)
- Register: hash password with bcrypt, save user
- Login: compare password, sign JWT (payload: id, role), return token
- Protect routes: middleware reads Authorization header, verifies JWT, sets req.user
- Frontend: store token in localStorage; include Authorization header on API calls

13) File upload (backend)

14) Auto-send email on verify (server snippet summary)
